Backing Up WordPress, Its a Dirty Job, but You Still Have to Do It

Jun 22nd, 2012

Famous last words, darwin awards, not things you want associated with your blog or website. Things should work smoothly, and always just work! unfortunately, that’s not how the internet works. Everything that can possibly break, sooner or later will. To make sure you survive, make backups of your WordPress install!

I hate reading, what should I do?

For small sites, install Backup WordPress, backup to a gmail account, easy and free backup. When you outgrow this plugin go for BackWPup or, a bit simpler to use: WordPress backup to Dropbox and do your backups to dropbox.

What to backup?

WordPress consists of:

  • WordPress itself – no need to backup, can be downloaded from WordPress.org
  • Themes – important to have the latest copy of your theme (the other themes are not important)
  • Upload – your images etc (note: This can be changed in Admin Panel > Settings > Media > Store uploads in this folder)
  • Your database
  • wp-config.php – in the root of your WordPress install (note: Sometimes wp-config.php is stored one directory higher, ask your developer)
  • .htaccess (in the root of your WordPress install)
  • Miscellaneous other files (ask your developer)

Some of these files almost never, if ever, change but you should have at least one copy.

To make a copy of your files by hand, use an ftp-client. Your database can be exported under Admin Panel > Tools > Export, or you can use phpMyAdmin.

Further reading on the WordPress codex:

  • Backing up your WordPress files
  • Manual backup resources
  • Using phpMyAdmin

Automating your backups

Doing things by hand is error-prone, this should be automated. For that, there are a plethora of plugins. Some good, some not so much. Here is a list of the once I like:

Free backup plugins

Free plugins are nice, easy to try, if you don’t like ‘m, no harm done. The down-side is that you get what you pay for. The free plugins can do the job, but you must pay attention and set things up right. These are the best plugins I have worked with:

  • WP db-manager – Allows you to optimize database, repair database, backup database, restore database, delete backup database , drop/empty tables and run selected queries. Supports automatic scheduling of backing up, optimizing and repairing of database.
  • Backup WordPress (git repo) – Backs up your entire site including your database and all your files once every day, can be send by mail, good for smaller installs
  • BackWPup – Complete backup solution, saves locally, dropbox etc.
  • wp Time Machine – Creates an archive of all your WordPress Data & Files and then stores them on Dropbox, Amazon’s S3, or your FTP host.
  • WordPress backup to Dropbox (git repo) [Main site][wpb2d] – straight forward backup to a dropbox account.
  • Xcloner – This plugin has the best restore function build in, but making backups is a bit more involved.

Complete list of the backup plugins on WordPress.org

What plugin to use? For small sites I would go with Backup WordPress, backup to a gmail account, easy and free backup. When you outgrow this plugin go for BackWPup or, a bit simpler to use: WordPress backup to Dropbox (but some functionality, like email notifications and zip support are premium extensions)

WP db-manager is often recommended, but it only backs up the database, I prefer the afore-mentioned plugins. They are simpler to use in the long-run.

If you don’t have a dropbox account, get one here for free (affiliate link, gives me free space).

Premium backup plugins

Considering how important backups are, you might want to go for a premium plugin. These are the ones most recommended:

  • Vaultpress – $15/month, from the makers of WordPress.com especially nice is their automatic restore.
  • Blog Vault – $9/month, automatic backup of your WordPress site, plugins and themes with blogVault.
  • Backup Creator – $7 about the same functionality as Backup WordPress (walkthrough)
  • BackupBuddy – $75 very powerful plugin, complete backups, database only, save to dropbox, ftp etc. (walkthrough)
  • Backuppress – $5/month Simple (to use) complete backup solution ([buprv][review])

Is it worth it to go premium? It is possible to do the same thing with a free plugin, but it is easier with a premium plugin. And what is definitely easier is restoring from backup, which is not the case using the free plugins.

Alternative ways of backing up your site

  • Noupe – Explains how to use cPanel to make a backup of your site, and phpMyAmin for the database.
  • Backuper – WP Backup plugin to be used with cPanel.
  • expand2web – Mentions blue host, which has its own WP backup solution

These kind of backup solutions are nice as an extra, or if you have very specific needs, but it is best to go for a free or premium backup plugin.

Backup and Restore,

Backup and restore are two sides of the same coin. One really does not exist without the other. To be sure that you are safe you have to do both at least once.

Don’t restore over your existing installation! Test on a test-machine (even if it is your desktop), or in a custom location (http://mysite/testwp/ for example)

The premium plugins have restore build-in. With the free plugins you have to do it yourself. The codex explains how to do it. If you want to move your install, meaning you want to change the url of your site read this on the codex and you might want to use the velvet blues update plugin, to fix links and locations.
